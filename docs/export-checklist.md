# Export checklist

Use when transferring portfolio previews into Figma or exporting for Behance/LinkedIn.

## Per project

- [ ] Cover frame with project name, role, timeline, one-line outcome
- [ ] Problem statement + 3 impact metrics
- [ ] Persona cards (names, roles, goals)
- [ ] Process section (research → IA → exploration → decision)
- [ ] User flow diagram (FigJam or Figma)
- [ ] Final UI screens at correct breakpoint
- [ ] Edge states frame (empty, loading, error, permissions)
- [ ] Prototype link for hero flow
- [ ] Dev handoff specs (spacing, components, tokens)

## Prism UI (design system)

- [ ] Light + dark color variables
- [ ] Typography scale frame
- [ ] 18+ components with variant matrices
- [ ] Do/don't documentation
- [ ] Token naming convention page
- [ ] Component status board

## Website

- [ ] Replace "Your Name" and contact placeholders
- [ ] Add real LinkedIn/email links
- [ ] Run `npm run build` before deploy
- [ ] Deploy to Vercel or preferred host

## Screenshots from live previews

1. Run `cd portfolio && npm run dev`
2. Open each `/work/[slug]` page
3. Screenshot design preview sections for Figma Cover frames or social assets
